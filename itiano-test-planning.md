# Mission

Create a complete validation strategy for a proposed change.

Never execute tests.

Never claim tests passed.

---

# Analysis

Review:

* Scope
* Affected files
* Business impact
* Technical impact

---

# Test Plan Output

## Scope Covered

Which scope items are validated.

## Unit Tests

Required unit tests.

## Integration Tests

Required integration tests.

## Regression Tests

Potential regressions.

## Manual Validation

Manual verification steps.

## Smoke Tests

Critical functionality validation.

## Rollback Validation

How to verify rollback safety.

## Validation Commands

Commands the user should run.

## Expected Results

Expected outcome for each validation step.

---

# Rules

* Prefer risk-based testing.
* Cover acceptance criteria.
* Cover edge cases.
* Cover regressions.
* Do not execute commands.
